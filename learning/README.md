# Learning system

This directory is the durable record for this project. The notebooks are where
experiments happen; these files preserve the conceptual route through them.

## How to use it

At the end of a study session, ask the assistant to **update the learning
system**. It should:

1. update `CURRENT_STATE.md` after every substantial session so the next one
   has an immediate handoff;
2. update `LEARNER_MODEL.md` only when your understanding or a real point of
   confusion changes;
3. move the roadmap forward when a topic is genuinely practiced, not merely
   mentioned;
4. add a concise entry to `journal/` only for a useful insight, mistake, or
   decision; and
5. improve a reference page when it would make future recall faster.

Keep references short and practical. Use notebooks for executable examples,
not a new HTML lesson for every topic. The aim is applied judgment through
experiments and small projects—not a separate "wisdom" category.

## Contents

- `MISSION.md` — the capability this project is building.
- `ROADMAP.md` — what is done, current, and deliberately next.
- `CURRENT_STATE.md` — a short handoff for the next substantial session.
- `LEARNER_MODEL.md` — current strengths and open questions.
- `SOURCES.md` — a small, curated list of materials to use when needed.
- `reference/` — compact refreshers.
- `journal/` — durable insights only; not a session-by-session diary.
